Agentic Coding with Claude Week 1 · Objective M05

Whose Decision Was It?

Reading a session transcript and attributing every decision — human or agent — is the skill your A1 reflection is graded on. Here you'll practice on a worked example.

M05 · Judgment Bloom: Analyze Assessed by: A1 reflection

Objective: Identify, in a completed session transcript, which decisions were made by the human and which by the agent.

The analysis frame

From objective M02 you have the vocabulary: Anthropic's session research found humans making ~70% of planning decisions (“what to do”) and Claude making ~80% of execution decisions (“how to do it”). To analyze a transcript, walk through it turn by turn asking three questions about each decision you spot:

  1. What was decided? Name it concretely (“use a JSON file for storage,” not “technical stuff”).
  2. Who decided? The one who chose, not the one who typed. An agent proposing and a human approving is a shared decision — note who had the veto.
  3. Which kind? Planning (what/whether/when to stop) or execution (how/where/in what order).

Decisions hide in unexpected places. Silence is a decision (the human who doesn't object to a proposed approach has approved it). Tool choices are decisions (the agent picked which test command to run). And “done” is always a decision — someone decided the work was finished.

A worked transcript

Below is a condensed but realistic Week-1-style session: a student building a small CLI utility (the A1 task). Read it once straight through, then work the analysis questions beneath it.

Human — turn 1

Build a CLI tool called wordcount that takes a filename and prints lines, words, and characters. Python, standard library only. Add a simple test.

Claude — turn 2

I'll create wordcount.py using argparse for argument handling, and a test_wordcount.py using unittest since you asked for standard library only. Reading the repo first to check for existing conventions…

[reads CLAUDE.md, lists directory] The repo is empty apart from README and CLAUDE.md. Creating both files now.

[permission prompt: create wordcount.py — human approves]

[permission prompt: create test_wordcount.py — human approves]

Claude — turn 3

Running the tests: python -m unittest … 1 failure — my character count includes the trailing newline, but my test expected it excluded. I'll make the implementation count all characters as-is (matching wc -c behavior) and fix the test expectation to match.

[edits test, reruns: 2 passed] Done — both tests pass.

Human — turn 4

Wait — I'd rather it not count the trailing newline. Most of my files end with one and it's confusing. Change it back.

Claude — turn 5

Understood — excluding a single trailing newline from the character count. Updated the implementation and the test; both pass. [shows diff]

Human — turn 6

What files have I changed? …OK, I've read the diff. Commit with message “Add wordcount CLI with tests”.

Claude — turn 7

[permission prompt: git add / git commit — human approves] Committed: Add wordcount CLI with tests.

Analyze it

Each question names a decision from the transcript. Classify it.

1. “The tool counts lines, words, and characters, in Python, standard library only.” Who decided, and what kind of decision?

A. Scope and constraints are the archetypal planning decision, and here they were fully specified by the human before the agent acted. Writing code that fulfills a specification is execution, not deciding the specification.

2. “Use argparse and unittest.” Who decided, and what kind?

C. “Standard library only” ruled options out but named nothing; several stdlib approaches existed (sys.argv, getopt, doctest…). Claude picked. That's a how-decision inside a human-set boundary — the everyday texture of the 80% execution share.

3. In turn 3, the trailing-newline question got its first answer (“count it, like wc -c”). Who made that decision, and what does turn 4 reveal about it?

B. The spec was silent on trailing newlines, so when the test failure forced the question, Claude resolved it — the agent made a decision about what the product does, not just how. Turn 4 is the human noticing and overriding: the division of decisions is a dial, and this is the dial being turned. Spotting exactly these moments — agent decisions that leak into “what” territory — is the heart of a good A1 reflection.

4. “The work is finished and worth committing.” Who decided?

D. Claude asserted completion in turn 3 — and was wrong (turn 4 reopened the work). Acceptance happened in turn 6, after the human read the diff. “Done” is a planning decision, and in a well-run session it belongs to the human. Note who typed the commit (agent) versus who decided to commit (human): typing isn't deciding.

5. Tally the transcript in the study's terms. Which summary is most accurate?

C. That's the humans-plan / agents-execute split with one instructive exception — the newline behavior started as an agent decision and was reclaimed by the human. Attribution is not only possible, it's the exercise: every decision has an owner, even shared ones (owner of the veto).

Now do yours

For Assignment A1's five-sentence reflection, apply this frame to your own completed session:

  1. Scroll back through the transcript (or run claude -c to reopen it).
  2. List every decision you can find — aim for at least six. Include the quiet ones: approvals, defaults you didn't question, “done.”
  3. Mark each Human or Claude, planning or execution.
  4. Find at least one decision Claude made that you'd call a what-decision (like the newline call). That one belongs in your reflection.
  5. Write the five sentences from the list — specific decisions, named owners.
Grader's-eye view

Weak reflection: “I made the big decisions and Claude handled the details.” Strong reflection: “Claude decided the character count should include the trailing newline when my spec was silent; I caught it in the diff and reversed it.” Specific decisions, named owners, and one moment where the boundary moved.
