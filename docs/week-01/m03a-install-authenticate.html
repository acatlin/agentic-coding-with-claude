Install & Authenticate

Get Claude Code onto your machine and signed in with your Pro account. Fifteen minutes, one command, one login.

M03 · Setup Bloom: Apply Assessed by: Assignment A1

Objective: Install Claude Code, authenticate with a Claude Pro account, and initialize a project with /init inside a Git repository. This lesson covers install + authenticate; Lesson 2 covers the project and /init.

Before you begin

You need three things, all from the course's tooling list:

  • A terminal: Terminal on macOS, PowerShell on Windows, any shell on Linux. (VS Code's built-in terminal works too.)
  • Your Claude Pro subscription — Claude Code authenticates against it directly; no API key needed.
  • About 15 minutes.

Step 1 — Run the installer for your OS

The official docs recommend the native install. Pick the one command that matches your system:

macOS, Linux, or WSL 
$ curl -fsSL https://claude.ai/install.sh | bash
Windows — PowerShell 
PS C:\> irm https://claude.ai/install.ps1 | iex
Windows — CMD 
C:\> curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
Windows: which shell am I in?

The docs call out the two classic mix-ups. If you see The token '&&' is not a valid statement separator, you're in PowerShell but ran the CMD command. If you see 'irm' is not recognized, you're in CMD but ran the PowerShell command. Check your prompt: it starts with PS C:\ in PowerShell and plain C:\ in CMD.

Windows only: install Git first

The docs recommend Git for Windows on native Windows so Claude Code can use its Bash tool (without it, Claude falls back to PowerShell as its shell). You need Git anyway for objective M06 — install it before Claude Code and you'll never think about it again.

Prefer a package manager? brew install --cask claude-code (macOS) and winget install Anthropic.ClaudeCode (Windows) also work — but note that unlike the native installer, they do not auto-update; you must run brew upgrade / winget upgrade yourself.

Step 2 — Confirm the install

$ claude --version

Per the Quickstart, a successful install prints a version number followed by (Claude Code). If the command isn't found, open a new terminal window first — the installer updates your PATH, but already-open terminals don't see it.

Step 3 — Log in with your Pro account

Claude Code prompts you to log in the first time you start a session:

$ claude

Follow the prompts: they open your browser, you sign in to your Claude account, and the credential is stored on your machine. The docs note that once logged in, you won't need to log in again — and if you ever need to switch accounts, type /login inside a running session.

Choose the subscription, not an API key

The login screen offers several account types. Choose Claude account (Pro) — that's what this course budgets around. A Console/API login bills per-token instead of using your flat $20/month Pro plan, and Week 8's economics exercises assume Pro usage limits.

Troubleshooting quick hits

  • Install command fails with a curl/403/syntax error: the docs point to Troubleshoot installation, which matches error text to fixes and lists alternative install methods.
  • Browser never opens during login: copy the URL printed in the terminal into your browser by hand.
  • Stuck? Ask in the cohort channel with the exact error text — this is also good practice for how you'll soon be prompting Claude.

Hands-on checklist

This objective is assessed by doing (Assignment A1). Work through each item in your own terminal and tick it off:

Self-check

1. On Windows, you run irm https://claude.ai/install.ps1 | iex and get 'irm' is not recognized as an internal or external command. What went wrong?

B. irm is a PowerShell alias; CMD doesn't know it. Check the prompt: PS C:\ means PowerShell, plain C:\ means CMD — each has its own installer one-liner.

2. Why does this course have you authenticate with your Pro subscription rather than an API key?

C. Claude Code accepts both, but the course is designed around Pro's flat $20/month and its usage limits — working within them is a stated learning outcome, and Week 8 studies the economics explicitly.

3. Install succeeded, but claude --version says “command not found” in the same terminal window. Most likely fix?

D. The installer modifies your PATH, but shells read PATH at startup — an already-open window won't see the new command. New terminal, then retry.
