Agentic Coding with Claude

Learning pages for the ten-week opt-in course on working effectively with AI labor: software work performed by an agentic coding tool under human direction.

Across ten weeks you will practice the three activities that determine whether delegated work succeeds — specification (grill sessions, PRDs, issue decomposition), direction (context management, plan-first and test-first execution), and review (diff review, code review, verification). You will ship one small application built spec-first, and leave with a Personal Agentic Workflow Playbook.

How to use these pages

Each week has a hub page listing that week's mastery objectives, with one or more learning pages per objective. Every learning page is self-contained — you can reach the objective from the page alone — and ends with a self-check matched to the objective's level. The official readings and videos are linked from each page if you want to go deeper.

Weeks

  • Week 16 objectives
    Orientation: What Is AI Labor?

    The agentic loop, tools and permissions, agentic coding vs. chat assistance, installing Claude Code, session controls, and your first GitHub repository.

  • Week 26 objectives
    Context: The Smart Zone

    Why context occupancy degrades output, /clear over /compact, writing a lean CLAUDE.md, scopes and skills, plan mode and rewind, and the fresh-vs-polluted session experiment.

  • Week 36 objectives
    Specification I: Grill Sessions & PRDs

    Alignment before implementation: grill sessions that interview you until the design is shared, PRDs as destination markers, expected doc rot, peer spec critique, and verifiable targets.

  • Week 46 objectives
    GitHub for Agentic Work

    Branches, PRs, and issues as units of agent work: decompose your PRD into vertical-slice tracer bullets, sequence the backlog to de-risk, file it with Claude + gh, and adopt one issue per session.

  • Week 56 objectives
    Execution: Plan-First & Test-First

    The agent writes production code under two disciplines: Explore–Plan–Code–Commit with a plan-approval gate, and red–green TDD — plus permission modes matched to the task, one fresh session per issue, and the interrupt-or-let-finish judgment.

  • Week 66 objectives
    Review: Owning the Diff

    Owning what the agent wrote: /diff's full and per-turn views, the four-checkpoint workflow, the six failure modes of AI-generated code, /code-review findings read critically, actionable review comments, and review depth proportionate to supervision.
