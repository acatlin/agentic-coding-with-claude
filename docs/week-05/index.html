Week 5 — Execution: Plan-First & Test-First

Specs and backlog in hand, this is the week the agent writes production code — under two disciplines that keep you in charge of what gets written. Plan-first: Claude reads, plans, and implements only after you approve. Test-first: one failing test at a time, because the feedback loop's quality is the ceiling on the agent's output. Around them: permission modes matched to the task, one fresh session per issue, and the judgment of when to grab the wheel.

Estimated total time: 4.0–5.0 hours (readings/videos + quiz + assignment + meeting). These pages cover the same ground as the assigned readings — use them as your primary lesson text, and the linked sources to go deeper.

This week's learning pages

Six mastery objectives, seven lessons (M25 spans two; mechanics M30 comes before judgment M29), in recommended order:

  • M25Apply
    Explore and Plan Before You Code

    Why jumping straight to code solves the wrong problem, plan mode as the enforcement, steering exploration, and the plan-approval gate with its four options.

  • M25Apply
    Code Against the Plan, Then Commit

    Implementation anchored to the approved plan, checks that close the loop, evidence over assertions, adversarial review, and the commit-and-PR handoff.

  • M26Apply
    Choose the Right Permission Mode

    The mode table from default to bypassPermissions, Shift+Tab, protected paths, auto mode's classifier — and justifying the mode by risk and review point.

  • M27Apply
    Failing Tests First: The Feedback Loop Is the Ceiling

    Red–green–refactor as agent direction: one failing test, observed red, minimal green — and why the feedback signal caps output quality regardless of model.

  • M28Apply
    One Issue, One Fresh Session, One PR

    The operating procedure for A5: fresh session seeded with CLAUDE.md + issue body, the full workflow chain, one PR with a closing keyword, worktrees for parallel runs.

  • M30Understand
    Course-Correcting a Running Agent

    The mid-run toolkit — Esc, redirect, rewind, “undo that,” /clear — what each changes, checkpoint limits, and how during-the-run differs from post-hoc review.

  • M29Evaluate
    Interrupt, or Let It Finish?

    The judgment call: direction errors vs. unfinished middles, criteria for each lean, the plan gate as the cheapest interrupt, and three worked calls from one session.

Suggested path

Read M25a and M25b together — their hands-on exercises are one workflow split at the approval gate. M26 and M27 each add one discipline to that workflow; do M27's single red–green cycle before starting A5 in earnest. M28 is A5's procedure — its checklist is the assignment's first issue. Finish with M30 then M29 (mechanics before judgment); both feed the quiz and Meeting 5's discussion.

How this week is assessed

  • Quiz 5 (25 MCQs): Explore–Plan–Code–Commit; permission modes and when each fits; red–green TDD with an agent; interrupt/redirect vs. post-hoc review. Every lesson's self-check is written in the quiz's application-over-recall style.
  • Assignment A5 (15 pts): implement two issues, each in its own fresh session seeded with CLAUDE.md and the issue body. For each: have Claude plan first and approve the plan; require failing tests before implementation; confirm green; open one PR per issue. Submit both PR links with a note on where you intervened.
  • Meeting 5: discussion of interrupt-vs-let-finish judgment (M29) — bring your intervention notes; expect “make the call” scenarios applied to your own sessions.
