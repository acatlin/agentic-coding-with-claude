Week 3 — Specification I: Grill Sessions & PRDs

Weeks 1–2 taught you to run and steer sessions; this week you learn to aim them. Alignment before implementation: get grilled by the agent until the design concept is shared, write it down as a 1–2 page PRD that marks the destination, critique a peer's spec with engineering lenses, and understand why verifiable targets beat open-ended prompting.

Estimated total time: 3.5–4.5 hours (readings/videos + quiz + assignment + meeting). These pages cover the same ground as the assigned readings — use them as your primary lesson text, and the linked sources to go deeper.

This week's learning pages

Six mastery objectives, six lessons, in recommended order:

  • M13Understand
    Why Grill First: Alignment Before Implementation

    What a grill session is — the six-sentence skill, quoted whole — and the failure modes it prevents, starting with good code that solves the wrong problem.

  • M14Apply
    Run a Grill Session

    The A3 procedure: install or emulate the skill, hold up your half of a 40-question interview, capture decisions as they resolve, and stop only at an accurate restatement.

  • M15Create
    Write the PRD

    Synthesize the grill session — no second interview — into goals, non-goals, user stories, constraints, and acceptance criteria that pass the stranger test. Commit it.

  • M16Understand
    PRD as Destination, Not Compiler Input

    Two models of a spec, why polish past unambiguity changes nothing about the build, and why doc rot is expected, acceptable, and designed for.

  • M17Evaluate
    Critique a PRD

    Three lenses — ambiguity, missing acceptance criteria, scope creep — a worked critique of a flawed sample PRD, and the protocol for delivering it at Meeting 3.

  • M18Understand
    Give the Agent a Target

    “Claude stops when the work looks done” — how a runnable check closes the loop, and why your acceptance criteria are targets written in prose.

Suggested path

Read M13, then run your grill session with M14 open — and write the PRD with M15 in the same sitting, while the session context is still warm. Read M16 before you're tempted to polish, and M17 before Meeting 3. M18 pairs naturally with writing your acceptance criteria; read it any time, but before the quiz.

How this week is assessed

  • Quiz 3 (25 MCQs): purpose and mechanics of grill sessions; PRD-as-destination; expected doc rot; targets and acceptance criteria. Every lesson's self-check is written in the quiz's application-over-recall style.
  • Assignment A3 (10 pts): choose your course project. Install or emulate the grill-me skill and run a grill session until Claude can restate your design concept accurately. Produce a one-to-two page PRD.md (goals, non-goals, user stories, constraints, acceptance criteria) and commit it. Bring it to the meeting for structured peer critique.
  • Meeting 3 exercise: structured critique of a peer's PRD using the three lenses from M17.
