Week 4 — GitHub for Agentic Work

Week 3 gave you a PRD; this week gives it wheels. Branches contain agent work, pull requests pause it for review, and issues become the unit you hand to a session: 6–10 vertical-slice tracer bullets with acceptance criteria, sequenced so the early ones test your riskiest assumptions, filed with Claude and the gh CLI — every draft reviewed before it's published.

Estimated total time: 3.0–4.0 hours (readings/videos + quiz + assignment + meeting). These pages cover the same ground as the assigned readings — use them as your primary lesson text, and the linked sources to go deeper.

This week's learning pages

Six mastery objectives, seven lessons (M19 spans two), in recommended order:

  • M19Apply
    Branches and Merges

    Branches as contained areas for agent work, the branch-per-issue convention, three merge methods and when each fits, and protected branches as enforcement.

  • M19Apply
    The Pull Request Lifecycle

    From branch to merge: the five-step lifecycle, draft PRs as “agent-done, not human-reviewed,” opening PRs with Claude, and the session-to-PR link.

  • M20Apply
    Issues as Units of Agent Work

    Titles that name behavior, bodies with checkbox acceptance criteria, the ready-for-agent label, closing keywords, and the issue→branch→PR→auto-close loop.

  • M21Create
    Tracer-Bullet Decomposition

    Vertical slices vs. horizontal layers, the three slicing rules, a worked 7-issue backlog for the flashcards PRD, and the expand–contract exception.

  • M22Evaluate
    Sequence the Backlog to De-Risk

    The frontier, riskiest-assumptions-first, why demo value isn't information value, and keeping priority decisions out of the blocking edges.

  • M23Apply
    File Issues with Claude + gh CLI

    Draft → review every draft for accuracy → file blockers-first with gh, ending with evidence (gh issue list) that the board matches the plan.

  • M24Understand
    One Issue per Session

    The week's converging rule: one issue, one fresh session, one branch, one PR — argued once from the smart zone and once from the reviewer's chair.

Suggested path

Read M19a and M19b together and do their checklists in one sitting — each drill feeds the next. M20's issue-to-close loop completes the mechanics. Then the A4 arc: decompose with M21, sequence with M22, and file with M23 — ideally decomposition and sequencing in one sitting while your PRD is fresh. Read M24 any time; it's short, it's on the quiz, and it's the reason next week's workflow looks the way it does.

How this week is assessed

  • Quiz 4 (25 MCQs): branch/PR lifecycle; anatomy of a good issue; vertical slices vs. horizontal layers; why one issue per session. Every lesson's self-check is written in the quiz's application-over-recall style.
  • Assignment A4 (10 pts): decompose your PRD into six to ten vertical-slice issues, each with acceptance criteria. Sequence them so early slices de-risk the design. Use Claude Code with the gh CLI to draft and file them — reviewing every draft before submission — and adopt a branch-per-issue convention. Submit the issue board link.
  • Meeting 4: bring your board; expect the M22 sequencing questions (“what does this slice's failure teach you?”) applied to your own backlog.
